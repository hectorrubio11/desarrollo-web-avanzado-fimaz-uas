Para esta práctica, creamos también una clase Usuario y dos clases hijas: Admin y Alumno.

Admin solo implementa el método getRol, y alumno implementa getRol y un valor matrícula, con su set y get respectivo.

En este caso, la clase Usuario maneja un verificador de correo, que devuelve una excepción en caso de que el correo no sea válido, el cual se imprime en el index, manejandolo con un Try/Catch.
